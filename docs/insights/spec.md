# 规格说明书 (Spec): 大语言模型 API 竞品分析系统

本项目是针对“字节跳动集团信息系统 AI 全栈项目挑战赛”量身定制的多 Agent 协同 LLM API 竞品分析系统。系统旨在通过多 Agent 自动化协作，完成对全球及中国主流 LLM 服务商（如 OpenAI, Anthropic, 豆包等）的核心指标（Pricing, Rate Limits, Latency, Features）的抓取、合规治理、结构化提取与对比报告生成。

---

## 1. 用户故事与目标 (User Story & Goal)

*   **作为**：企业 AI 平台架构师或产品团队负责人，
*   **我希望**：自动化地监控和对比市场上各大主流 LLM 服务商的最新的 API 报价、限流规则和功能特性，
*   **以便于**：为公司的 AI 应用选型提供最具性价比、合规且权威的数据支持，摆脱手工收集、信息分散且时效性差的痛点。

---

## 2. 功能性需求与验收准则 (Acceptance Criteria)

### 场景一：多 Agent 并发采集与合规脱敏
*   **Given (前置条件)**：系统配置了要监测的竞品服务商列表（例如 OpenAI, 豆包）以及数据抓取源（真采/Mock 双模）。
*   **When (动作触发)**：用户发起竞品分析任务。
*   **Then (预期结果)**：
    1.  **采集 Agent** 并发启动，抓取各大竞品网站的 API 价格页和开发者文档。
    2.  数据流经 **脱敏/合规 Agent**，对文本中可能包含的个人敏感信息（如测试手机号、敏感 API Key 凭证、个人访谈数据）进行正则/LLM 双重遮蔽，产出脱敏后的安全语料。

### 场景二：基于强 Schema 的结构化知识抽取与质检打回闭环
*   **Given (前置条件)**：已定义严格的竞品知识 JSON Schema（详见 Design 说明）。
*   **When (动作触发)**：分析师 Agent 提取脱敏后的语料并进行结构化整理，随后将其提交给质检 Agent。
*   **Then (预期结果)**：
    1.  **分析师 Agent** 输出包含定价（每百万 Token 单价）、限流（TPM/RPM）、支持的上下文长度、以及是否支持 Function Calling 的结构化 JSON 数据。
    2.  **质检 Agent** 进行 Schema 强校验与幻觉抑制检查：
        *   *若校验通过*：流转至报告撰写阶段。
        *   *若校验不通过（如数据字段残缺、溯源缺失）*：**触发物理打回**，将不足之处（如“OpenAI 价格缺少 Prompt/Completion 细分单价”）作为反馈消息派发回分析师 Agent 重新抽取，最多迭代 3 次，形成闭环。

### 场景三：报告撰写与信息溯源 Trace 展示
*   **Given (前置条件)**：结构化 JSON 数据通过质检校验。
*   **When (动作触发)**：报告撰写 Agent 开始渲染最终报告。
*   **Then (预期结果)**：
    1.  生成精美的 Markdown/PDF 竞品对比报告。
    2.  报告的每一个分析结论（如“豆包 API 价格比 OpenAI 便宜 90%”）必须强制绑定**信息溯源（Source Attribution）**，标注具体的数据来源（如：`Source: [Doubao API pricing page](http://...)`），并在前端支持可观测 Trace 详情查阅。

---

## 3. 非功能性需求与性能指标 (NFR)

1.  **可观测性 (Observability)**：必须集成类似 LangSmith 的 Trace 机制，记录多 Agent 交互的完整 DAG DAG Trace，支持可视化运行过程与消息传递。
2.  **合规与安全 (Compliance & Privacy)**：
    *   采集到的敏感测试数据（如个人 API Key）在分析前必须 100% 脱敏，避免敏感信息泄露。
    *   保证采采集合规，不侵犯数据隐私。
3.  **易用性与部署性 (Usability & Deployment)**：
    *   **一键 Docker 部署**：支持前后端 + Mock 服务的 Docker-compose 一键拉起，无需繁琐安装本地依赖。
    *   **前端页面展示**：提供直观的 Web 交互，可视化展示多 Agent 协同打回的动态过程及竞品对比结果。
